The UI Settings page can be used to change the Theme hues (colors) of the dashboard, load a different Logo, and set the Code coloring.




Custom Logo

If you would like to use your own logo, you can upload the file by clicking the paper clip icon to open the file store dialog. Once selected the image path should display in the Logo source input and you should save the UI settings to see the dashboard logo update. We recommend using a .png or .svg image with a transparent background for the best look.


Custom UI Theme

If you choose the userTheme option in the Choose theme select, you can choose from the 20 Material Design Color Palettes to use within the Mango UI.

Per the Material Design Color Spec, you choose one color palette for each of the three color intentions plus one for the background palette.

  • Primary palette - Used to represent primary interface elements.
  • Accent palette - Used to represent secondary interface elements.
  • Warning palette - Used to represent interface elements that the user should be careful of.

The Mango UI will automatically update it's interface elements to use the color palettes, so remember to save before leaving the page!

Advanced users can modify ${MA_HOME}/web/modules/mangoUI/web/ui/uiSettings.json if they wish to take color customization to the next level. Using the method explained in this support article you can create your own custom color palettes and themes that utilize the exact colors you specify.

You can turn on Dark theme on any theme by checking the Dark theme option. A dark version of the background palette will be used and white text will display on top for maximum contrast.

Additionally you can set the Code coloring theme to be used across the dashboard. Choose a theme from the dropdown and you will see a live preview of the code coloring below. Once saved, the Code coloring will be used in Edit pages, Play area, and Examples pages.


Google API Settings

In addition you can also set your Google Maps API Key used for working the the <ma-map> component and a Google Analytics Property ID used for tracking visitor activity on your Mango Server.




User Module Settings

You may want to extend the dashboard tools you have available by writing your own AngularJS code. Using the User module URL feature you can add your own Angular components, directives, filters and controllers. Read the full support article to learn how to leverage writing your own Angular code to extend the application. You can also add your own custom .css file to the User stylesheet URL to apply to all your pages or override the dashboard UI.




Show unacknowledged events icon in toolbar

You can hide the icon in the toolbar and prevent the UI from making queries for unacknowledged events on every page load by disabling this setting. By default this is enabled.
